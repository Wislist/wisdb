# 测试文件说明

本文档用于说明当前后端核心测试文件的目标、覆盖范围和编写约定，便于后续扩展测试时保持一致。

## 适用范围

- `src/main/backend/dm/data_manager_test.go`
- `src/main/backend/dm/pcacher/pcacher_test.go`
- `src/main/backend/utils/cacher/cacher_test.go`

## data_manager_test.go 说明

### 测试目标

验证 DataManager 在常见场景下的行为正确性，重点覆盖插入、读取、非法数据处理与并发场景。

### 已覆盖用例

- `TestDataManagerInsertRead`
  - 正常插入并读取数据
  - 校验读取内容与写入内容一致
- `TestDataManagerInsertTooLarge`
  - 传入超过页可用空间的数据
  - 期望返回 `ErrDataTooLarge`
- `TestDataManagerReadInvalid`
  - 将数据项标记为无效后读取
  - 期望 `ok=false`
- `TestDataManagerConcurrentInsertRead`
  - 多协程并发插入与读取
  - 校验读写一致性与基本并发稳定性

### 编写要求

- 每个测试用例只验证一个清晰行为。
- 用 `t.TempDir()` 生成隔离测试目录，避免互相污染。
- 读到 `Dataitem` 后在测试末尾调用 `Release()`，避免缓存引用泄漏。
- 并发测试使用 `WaitGroup` 和错误通道收敛异常。

## pcacher_test.go 说明

### 测试目标

验证页面缓存层的基本持久化、重开加载、截断行为及参数边界。

### 已覆盖用例

- `TestPcacherCreateGetReleasePersist`
  - 创建页面后读取校验
  - 修改脏页并释放，再次读取确认数据落盘
- `TestPcacherOpenAndTruncate`
  - 重启后页数恢复
  - 截断后页数和文件大小符合预期
- `TestPcacherCreateMemTooSmallPanics`
  - 内存不足时应触发 `ErrMemTooSmall` panic

### 编写要求

- 测试数据尽量使用固定构造，避免随机导致偶发失败。
- 涉及磁盘行为时同时校验内存视图与文件大小。
- 需要 panic 的场景，统一使用 `defer recover` 断言错误类型。

## cacher_test.go 说明

### 测试目标

验证通用缓存组件在并发访问、对象共享与引用归零释放下的行为。

### 已覆盖用例

- 并发同键读取返回同一底层对象
- 对象读写内容在共享实例上可见
- 引用归零后释放，再次获取触发重建

### 编写要求

- 测试需要显式调用 `Release(uid)` 保证引用计数回收。
- 并发测试避免使用非线程安全断言状态。

## 新增测试建议

- DM 的恢复链路：`Create -> Insert -> Open -> Recovery` 场景。
- DM 的事务状态分支：结合真实 `tm` 的提交/回滚状态验证恢复逻辑。
- Pcache 的高并发读写压测：验证页竞争下的稳定性。
